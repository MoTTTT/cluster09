Cluster API Cluster Provisioning helm chart: cluster-chart

Thank you for installing {{ .Chart.Name }}. Your release is named {{ .Release.Name }}. Cluster API resources have been created in the {{ .Values.cluster.name }} namespace.

To view progress with the Cluster API provisioning (run against the Management Cluster):
  $ clusterctl describe cluster {{ .Values.cluster.name }} -n {{ .Values.cluster.name }} --show-conditions all --show-templates --show-resourcesets --grouping=false --echo

To extract configuration secrets for the target cluster (run against the Management Cluster):
  
  # Extract Talos configuration
  $ kubectl get secret --namespace {{ .Values.cluster.name }} {{ .Values.cluster.name }}-talosconfig -o jsonpath='{.data.talosconfig}' | base64 -d > {{ .Values.cluster.name }}-talosconfig

  # Extract standard kubeconfig (Required for cluster owners or bastion access)
  $ kubectl get secret --namespace {{ .Values.cluster.name }} {{ .Values.cluster.name }}-kubeconfig -o jsonpath='{.data.value}' | base64 -d > {{ .Values.cluster.name }}-kubeconfig

NOTE: If accessing via a bastion server, edit {{ .Values.cluster.name }}-kubeconfig to point the 'server' field to your bastion/port or localhost:port if tunneling.

To activate the context for the new cluster using Talos:
  $ talosctl kubeconfig --nodes {{ .Values.controlplane.endpoint_ip }} --endpoints {{ .Values.controlplane.endpoint_ip }} --talosconfig=./{{ .Values.cluster.name }}-talosconfig

---
DASHBOARD & LOGS (Requires active Talos context)

To view the Talos dashboard:
  $ talosctl dashboard --nodes {{ .Values.controlplane.endpoint_ip }} --endpoints {{ .Values.controlplane.endpoint_ip }}

To view dmesg or service logs (e.g. machined):
  $ talosctl dmesg --nodes {{ .Values.controlplane.endpoint_ip }}
  $ talosctl logs machined --nodes {{ .Values.controlplane.endpoint_ip }}

GITOPS BOOTSTRAP

GitOps manifests (GitRepository and FluxInstance) are typically loaded automatically via Talos ExtraManifests. To complete the bootstrap, ensure the SOPS decryption key is available in the new cluster.

1. Add the SOPS age key (run against the target cluster):
  $ cat ~/.config/cluster-api/age.agekey | kubectl create secret generic sops-age --namespace=flux-system --from-file=age.agekey=/dev/stdin

Flux will detect the key on the next reconciliation cycle and begin pulling workload kustomizations.
